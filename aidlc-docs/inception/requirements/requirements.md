## プロジェクト概要

「だが、それでいい」は、「今日も何かできた」を見つけるための行動支援アプリケーション。従来の習慣トラッカーが「一つの習慣をストイックに継続させる」ことを目的とするのに対し、本アプリは目標通りにできた日を最高としつつも、別のことができた日も十分に価値があると捉える。

**コンセプト**: 0点か100点かではなく、何かできた日はみんな合格点。

### FR-11: Learning Engineによる最適化

**概要**: Amazon Bedrockを活用した行動パターン学習とRecommendation最適化

**行動モデルについて**: 行動モデルとは、ユーザーの行動パターンを表す内部データ構造。Action_Logから以下のデータを蓄積して構築する:

- **いつ行動するか** — 時間帯・曜日のパターン（例: 「平日夜に行動しやすい」）
- **何に反応するか** — Yes/No応答の傾向（例: 「筋トレ系は受け入れやすい、英語系は断りがち」）
- **どのくらいやるか** — actionLevel（normal/minimal）の傾向
- **どのGoalに動くか** — Primary vs Pivot の選択傾向

このデータをAmazon Bedrockへのコンテキストとして渡すことで、精度の高いRecommendationを生成する。

**Learning Engineの実行方式**: 週次バッチ処理（毎週月曜0時にAWS Lambda + EventBridgeで実行）。Action_Logへの記録はリアルタイムで行い、行動モデルの構築・更新はバッチで処理する。

| ID      | 要件                                                                                                                                                                                                                                                                                                                                                                                        |
| ------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| FR-11-1 | Action_Log（Yes/No応答・Done記録・時間帯・曜日・Goal種別・actionLevel）をリアルタイムで蓄積する。週次バッチ処理でAction_Logを解析し、上記の行動モデルを構築・更新する                                                                                                                                                                                                                       |
| FR-11-2 | Recommendation生成時にAction_Logが7件未満の場合はProfileのみを参照したデフォルトプロンプトを使用する。7件以上蓄積されている場合、直近の週次バッチで構築された行動モデルをBedrockへのコンテキストに加えたパーソナライズモデルを使用する。なお、Action_Logが大量に蓄積された場合にBedrockのコンテキスト上限を超えないよう、ログの要約・圧縮・件数制限などのコンテキスト管理処理をv2で実装する |
| FR-11-3 | 学習データのリセット要求時に確認ダイアログを表示した後にAction_LogとLearning_Engineモデルをリセットする                                                                                                                                                                                                                                                                                     |

## 用語集

| 用語                 | 定義                                                                                                                          |
| -------------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| DagaSoreDeIi_App     | 本アプリケーション「だが、それでいい」全体を指すシステム名                                                                    |
| User                 | アプリを利用する個人ユーザー                                                                                                  |
| Profile              | ユーザーの属性・興味・行動傾向・心理状態などを記録したデータ                                                                  |
| Goal                 | ユーザーが設定する目標。複数設定可能                                                                                          |
| Primary_Goal         | ユーザーが最優先として設定したGoal                                                                                            |
| Pivot_Goal           | Primary_Goalの達成が難しい日に提案される別の価値ある選択肢                                                                    |
| Flexible_Achievement | 当初の目標でなくても行動できたことを肯定的に評価する概念                                                                      |
| State_Detection      | ユーザーの現在状態を検知する機能                                                                                              |
| Trigger              | State_Detectionによって検知された行動提案を発火させる条件                                                                     |
| Recommendation       | TriggerをもとにAIが生成する行動提案                                                                                           |
| Pivot                | Recommendationを拒否/無視した際に別のGoalへ提案を切り替える機能                                                               |
| Effort_Point         | 行動完了時に付与されるポイント                                                                                                |
| Persona_Message      | Future_Self_Modelが語りかけるスタイルのメッセージ                                                                             |
| Learning_Engine      | 行動結果を蓄積しRecommendationを最適化するAIコンポーネント（Amazon Bedrock）                                                  |
| Action_Log           | ユーザーが実際に行った行動の記録                                                                                              |
| Similar_User_Data    | 類似プロフィールを持つ他ユーザーの匿名化された行動データ（v2以降）                                                            |
| Future_Self_Model    | Similar_User_Dataを基に構築した「未来の自分」モデル（v1: モックデータ）                                                       |
| Milestone            | 累計Effort_Pointが100の倍数に達したときに発生する達成イベント。特別な達成メッセージとバッジ表示をトリガーする（FR-08-7）      |
| Action_Ticket        | Recommendationへの応答時に生成される行動タスク（Open/Done）。content フィールドに具体的なアクションステップ（行動手順）を含む |
